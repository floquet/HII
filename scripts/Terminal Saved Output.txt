I also notice there's a warning about the daemon not using the default seccomp profile. This is a security-related warning - seccomp (secure computing mode) is a Linux kernel feature that restricts the system calls a container can make to the host system. If you're running in a development environment this might not be critical, but in a production environment you might want to investigate why it's not using the default profile.

Inside /var/lib/docker, you'll find several subdirectories:

    overlay2/ - where the actual image layers and containers are stored
    containers/ - container configuration files
    image/ - image metadata
    volumes/ - named volumes data
